## 摘要

随着互联网电商的快速发展，平台积累了海量的用户行为数据，如何从中提取有价值的信息并应用于精准营销和经营决策，已成为数据科学领域的重要研究方向。本文旨在设计并实现一个基于Python的电商用户行为分析与预测系统，以帮助运营人员更直观地理解用户行为规律，并实现对用户复购行为和优惠券核销行为的智能预测。

本系统以阿里巴巴O2O优惠券使用预测数据集（Online部分）为基础，数据时间跨度为2016年1月至2016年6月，涵盖超过544,307名活跃用户、11,429,826条行为记录。系统后端采用Python语言结合FastAPI框架构建RESTful API服务，使用MySQL数据库存储结构化数据，通过Pandas进行数据清洗与特征工程，构建了涵盖用户维度、商户维度、交互维度和时间维度的100余个特征；前端采用Vue 3框架结合ECharts可视化库，实现了总览仪表盘、用户行为分析、行为流转化、商户分析以及平销期与促销期对比等多个可视化模块。预测模块采用XGBoost与LightGBM融合模型，针对平销期和促销期（含618大促）分别构建预测模型，在测试集上用户复购预测AUC达0.81，优惠券核销预测AUC达0.79，模型性能良好。

**关键词：** 用户行为分析；数据可视化；机器学习；XGBoost；FastAPI

## ABSTRACT

DESIGN AND IMPLEMENTATION OF AN E-COMMERCE USER BEHAVIOR ANALYSIS AND PREDICTION SYSTEM BASED ON PYTHON

With the rapid development of Internet e-commerce, platforms have accumulated massive amounts of user behavior data. Extracting valuable information from such data and applying it to precision marketing and business decision-making has become an important research direction in data science. This paper aims to design and implement a Python-based e-commerce user behavior analysis and prediction system to help operators intuitively understand user behavior patterns and intelligently predict user repurchase behavior and coupon redemption behavior.

The system is based on Alibaba's O2O coupon usage prediction dataset (Online portion), covering data from January to June 2016, with over 544,307 active users and 11,429,826 behavioral records. The backend is built using Python with the FastAPI framework to provide RESTful API services, MySQL for structured data storage, and Pandas for data cleaning and feature engineering, constructing over 100 features across user, merchant, interaction, and temporal dimensions. The frontend is developed using Vue 3 combined with the ECharts visualization library, implementing multiple visualization modules including an overview dashboard, user behavior analysis, behavior flow conversion, merchant analysis, and a comparison between regular and promotional periods. The prediction module employs an XGBoost and LightGBM ensemble model, with separate models built for the regular period and the promotional period (including the 618 shopping festival). The user repurchase prediction achieved an AUC of 0.81 on the test set, while the coupon redemption prediction achieved an AUC of 0.79.

**Key words:** user behavior analysis; data visualization; machine learning; XGBoost; FastAPI

## 1 绪论

### 1.1 选题背景

近年来，随着移动互联网和电子商务的高速发展，电商平台积累了规模庞大的用户行为数据。用户在平台上的每一次点击、领券和购买行为，都蕴含着丰富的消费偏好信息。如何有效地挖掘、分析这些数据，已成为提升平台运营效率、优化用户体验的关键所在。

以阿里巴巴、京东为代表的国内头部电商平台，每年在"618""双11"等大促期间产生的交易数据量以亿级计。促销活动期间的用户行为模式与平销期存在显著差异：促销期间用户的购买转化率显著提升，领券核销率也随之变化，但与此同时，优惠券的滥发和用户的非理性消费也给平台带来了额外成本。因此，准确识别平销期与促销期用户行为规律的差异，对平台制定差异化的运营策略具有重要意义[1]。

在此背景下，构建一个能够对电商用户行为进行可视化分析并实现智能预测的系统，不仅有助于平台运营人员快速洞察数据规律，也为精准营销和优惠券发放策略的优化提供了数据支撑。本文以阿里巴巴O2O优惠券使用预测数据集为基础，设计并实现了一套完整的电商用户行为分析与预测平台。

### 1.2 选题意义

本研究的意义主要体现在以下几个方面：

（1）提升运营决策效率：通过对海量用户行为数据进行可视化展示，运营人员无需具备深厚的数据分析背景，即可直观地掌握用户活跃度、转化漏斗、RFM分层等核心运营指标，从而提升决策效率。

（2）优化优惠券投放策略：优惠券核销预测模块能够预测用户领券后是否会实际使用，帮助平台将有限的优惠券资源精准投放给高核销倾向的用户，减少资源浪费，提升投入产出比。

（3）支持精准营销：用户复购预测模型可识别出短期内有较高复购意愿的用户群体，为平台的召回营销和个性化推荐提供依据，有效提升用户留存率。

（4）揭示促销规律：通过平销期与促销期（618大促）的双场景对比分析，系统能够量化促销活动对用户行为的影响，为后续促销策略的设计提供数据参考[2]。

（5）推动大数据技术应用：本项目综合运用了数据处理、特征工程、机器学习、前后端分离开发等多项技术，是大数据技术在电商运营领域的一次完整实践，具有较好的参考价值。

### 1.3 国内外研究现状

#### 1.3.1 国外研究现状

用户行为分析的研究最早可追溯到Web日志挖掘领域。Cooley等人于1999年提出了基于用户浏览日志的行为模式挖掘方法，奠定了用户行为分析的理论基础。随着机器学习技术的成熟，深度学习方法逐渐被应用于用户行为序列建模。Google于2016年提出的Wide & Deep模型将线性模型与深度神经网络结合，在推荐系统和转化率预测中取得了优异效果[3]。在梯度提升树模型方面，Chen和Guestrin于2016年提出的XGBoost算法因其在结构化数据上的卓越性能，成为业界最广泛使用的预测模型之一。Ke等人于2017年提出的LightGBM则进一步提升了梯度提升树的训练效率，在大规模数据集上展现出明显优势[4]。

#### 1.3.2 国内研究现状

国内学者在电商用户行为分析领域也进行了大量研究。阿里巴巴在2016年天池竞赛中公开了O2O优惠券使用预测数据集，吸引了众多研究者针对优惠券核销行为构建预测模型，推动了该领域的研究发展。在可视化分析方面，百度推出的ECharts开源可视化库已成为国内数据可视化领域的主流工具，被广泛应用于各类大数据分析平台的前端开发[5]。近年来，随着Flask、FastAPI等轻量级Python Web框架的普及，越来越多的研究者和开发者选择将数据分析与Web应用开发相结合，构建端到端的数据分析平台。然而，目前针对电商用户行为的分析系统大多只关注单一场景，缺乏对平销期与促销期行为模式差异的系统性对比研究，本文的工作在一定程度上填补了这一空白。

### 1.4 设计内容

在充分调研相关技术和数据集的基础上，本文确定了系统的设计方案，主要包括以下五个方面：

（1）数据处理与特征工程：对阿里巴巴O2O数据集进行清洗与预处理，按照平销期和促销期的时间划分策略构建训练集、验证集和测试集，并从用户维度、商户维度、交互维度和时间维度构建100余个特征变量。

（2）多维度数据可视化：基于Vue 3和ECharts实现总览仪表盘、用户行为趋势图、RFM用户分层、转化漏斗、行为热力图等多类型可视化图表，支持多维度筛选和数据钻取。

（3）双场景对比分析：实现平销期与促销期（含618大促）用户行为模式的对比分析，从转化率、活跃度、优惠券使用等维度展示两个时期的差异。

（4）智能预测模型：分别针对平销期和促销期训练用户复购预测模型和优惠券核销预测模型，采用XGBoost与LightGBM融合方案，支持单用户预测演示、批量预测和特征重要性分析。

（5）系统集成与部署：基于FastAPI构建后端RESTful API服务，通过MySQL数据库统一管理数据，实现前后端分离架构的完整系统集成。

## 2 相关技术研究

### 2.1 系统技术简介

#### 2.1.1 Python语言

Python是一种高级的、通用的解释型编程语言，由Guido van Rossum于1989年设计开发，以简洁清晰的语法和强大的生态系统著称。Python在数据科学、机器学习和Web开发领域拥有极为丰富的第三方库支持，是目前数据分析类系统开发的首选语言。

Python语言的主要特点如下：

（1）语法简洁易读：Python采用缩进式代码块，语法接近自然语言，开发效率高，便于团队协作与代码维护。

（2）生态系统完善：Python拥有NumPy、Pandas、Scikit-learn、XGBoost、LightGBM等成熟的数据科学库，以及FastAPI、Flask等Web框架，可一站式满足数据处理、模型训练和后端服务开发的需求。

（3）跨平台性强：Python可在Windows、Linux、macOS等多个操作系统上运行，部署灵活。

（4）社区活跃：Python拥有全球最大的开源社区之一，文档完善，问题解决资源丰富。

本系统后端全程采用Python 3.9开发，涵盖数据处理、特征工程、模型训练与API服务等所有核心功能模块。

#### 2.1.2 FastAPI框架

FastAPI是一个基于Python的现代化高性能Web框架，由Sebastián Ramírez于2018年发布。相比传统的Flask框架，FastAPI具有以下显著优势：

（1）性能卓越：FastAPI基于Starlette和Pydantic构建，采用异步（async/await）处理机制，性能接近NodeJS和Go，在高并发场景下表现优异。

（2）自动生成API文档：FastAPI可根据代码中的类型注解自动生成符合OpenAPI规范的交互式文档（Swagger UI），极大方便了前后端联调。

（3）数据验证自动化：通过Pydantic模型对请求参数进行类型校验，减少手动校验代码，提升接口健壮性。

（4）生命周期管理：FastAPI支持通过`lifespan`上下文管理器在应用启动时预加载数据、关闭时清理资源，本系统正是利用此机制在启动时将特征数据加载至内存缓存，显著降低了接口响应延迟。

本系统使用FastAPI 0.104+版本搭建后端服务，通过`uvicorn`作为ASGI服务器启动，监听`127.0.0.1:8000`端口，并配置了CORS中间件以支持前端跨域访问。

#### 2.1.3 Vue 3前端框架

Vue 3是由尤雨溪开发的渐进式JavaScript前端框架，于2020年正式发布。Vue 3相比Vue 2在性能、TypeScript支持和代码组织方面均有显著提升，是目前国内前端开发的主流框架之一。

Vue 3的主要特点包括：

（1）Composition API：Vue 3引入了全新的组合式API，使得复杂逻辑可以按功能模块组织，代码复用性大幅提升。

（2）性能优化：Vue 3重写了虚拟DOM实现，渲染性能相比Vue 2提升约40%，同时内存占用更低。

（3）TypeScript原生支持：Vue 3使用TypeScript重写了核心代码，为开发者提供了完善的类型推断和编辑器支持。

（4）丰富的生态：配合Vite构建工具、Element Plus UI组件库和Pinia状态管理库，可快速构建功能完整的中大型前端应用。

本系统前端基于Vue 3.3+版本开发，采用TypeScript编写，使用Vite 5.0作为构建工具，通过Axios与后端FastAPI进行数据交互。

#### 2.1.4 ECharts可视化库

ECharts是百度公司开源的基于JavaScript的数据可视化库，目前已捐献给Apache软件基金会，成为Apache顶级项目。ECharts支持折线图、柱状图、饼图、散点图、漏斗图、热力图、桑基图等数十种图表类型，并提供丰富的交互功能和动画效果。

ECharts的主要特点如下：

（1）图表类型丰富：支持从基础统计图表到地图、关系图等高级可视化类型，完全满足本系统多维度数据展示的需求。

（2）交互性强：支持图例联动、数据缩放、区域选择、鼠标悬停提示等交互功能，提升用户数据探索体验。

（3）性能优秀：ECharts底层使用Canvas渲染，在大数据量场景下仍能保持流畅的渲染性能。

（4）与Vue集成便捷：配合`vue-echarts`等封装库，可将ECharts图表作为Vue组件使用，实现响应式数据绑定。

本系统使用ECharts 5.4+实现了转化漏斗图、RFM散点图、行为热力图、桑基图、双场景对比折线图等多种可视化图表。

#### 2.1.5 XGBoost与LightGBM模型

XGBoost（Extreme Gradient Boosting）是由陈天奇等人于2016年提出的梯度提升树实现，以其在结构化数据上的卓越性能和高效的并行计算能力著称。XGBoost在模型训练中引入了正则化项以控制过拟合，支持特征采样（`colsample_bytree`）和样本采样（`subsample`），在众多机器学习竞赛中取得了优异成绩。

LightGBM（Light Gradient Boosting Machine）是微软于2017年开源的梯度提升框架，相比XGBoost具有更快的训练速度和更低的内存占用。LightGBM采用基于直方图的决策树算法和叶子优先（Leaf-wise）的生长策略，在大规模数据集上的训练效率比XGBoost提升数倍至十倍以上。

本系统同时训练XGBoost和LightGBM两个模型，并采用加权概率融合策略（XGBoost权重0.4，LightGBM权重0.6）得到最终预测结果。此外，系统通过阈值寻优（遍历0.1至0.8区间）选取使F1-Score最大化的最优决策阈值，提升了预测结果的实用性。

#### 2.1.6 MySQL数据库

MySQL是目前最流行的开源关系型数据库管理系统之一，由瑞典MySQL AB公司开发，后被Oracle收购。MySQL以其高性能、高可靠性和易用性著称，被广泛应用于各类Web应用和数据分析系统。

MySQL的主要特点包括：

（1）高性能：支持高并发读写操作，具备完善的索引机制和查询优化器，适合存储和检索大规模结构化数据。

（2）事务支持：MySQL的InnoDB存储引擎支持ACID事务，确保数据一致性和完整性。

（3）SQL标准支持：支持标准SQL语言，便于进行复杂的多表关联查询和聚合统计。

（4）与Python集成便捷：通过SQLAlchemy ORM框架可与Python无缝集成，支持以Python对象的方式操作数据库表。

本系统使用MySQL 8.0存储用户行为原始数据、用户特征数据、商户特征数据和预测结果数据，通过SQLAlchemy 2.0进行ORM映射。

### 2.2 开发工具与开发环境

本系统开发过程中使用的主要开发工具如下：

（1）PyCharm：JetBrains公司开发的Python集成开发环境，提供智能代码补全、调试器、版本控制集成等功能，用于后端Python代码的编写与调试。

## 4 系统详细设计与实现

### 4.1 数据处理与特征工程

#### 4.1.1 数据清洗与预处理

原始数据集在导入系统前需经过一系列清洗步骤，以确保数据质量。主要处理内容包括以下几个方面：

首先对字段类型进行规范化处理，将`Date`和`Date_received`字段统一转换为`datetime`格式，便于后续时间窗口计算。其次处理缺失值：`Coupon_id`为null时表示普通消费行为，`Date`为null且`Coupon_id`不为null时表示领券未使用，这两种情况均属于正常业务逻辑，不做删除处理，而是通过标志位加以区分。对于`Discount_rate`字段，其存在"x:y"（满减）、小数（折扣率）和"fixed"（限时低价）三种格式，需统一转化为折扣率数值型字段以供模型使用。此外，对少量重复记录进行去重处理。

#### 4.1.2 时间窗口划分与标签构建

本系统针对平销期和促销期分别构建训练数据。平销期模型以2016年1月至3月的行为数据计算用户特征，以4月数据训练模型，以5月数据作为测试集；促销期模型以1月至5月数据计算特征，以4月和5月联合数据训练，以6月（含618大促）数据测试。

用户复购标签的构建逻辑为：在预测目标月份内，若用户产生至少一次购买行为（`Action=1`），则标记为正样本（`label=1`），否则为负样本（`label=0`）。

#### 4.1.3 特征工程

本系统从四个维度共构建100余个特征变量：

**用户维度特征（50+个）：** 基础行为统计特征（7天/14天/30天滑动窗口内的点击次数、领券次数、购买次数）、转化特征（CTR、领券率、券核销率、整体CVR）、活跃度特征（Recency、Frequency、平均行为间隔、行为间隔方差、生命周期长度）和行为模式特征（最常活跃时段、工作日与周末行为占比）。

**商户维度特征（20+个）：** 商户热度指标（总点击数、总购买数）、商户转化率、商户覆盖用户数及商户活跃排名。

**交互特征（15+个）：** 用户-商户历史交互次数、用户-商户转化率、是否为回头客以及最近交互距今天数。

**时间特征（10+个）：** 星期几、是否周末、是否月初/月末、距离618大促的天数。

**促销敏感度特征（8+个）：** 历史领券偏好、券弃用率和领券到购买的平均间隔天数。

## 5 系统测试

### 5.1 系统测试的意义

系统测试是软件开发流程中保证系统质量不可缺少的环节，通过对系统各功能模块进行全面、系统的测试，可以及时发现并修复潜在问题，确保系统在实际使用中的稳定性、准确性和可靠性。

对于本系统而言，系统测试的意义具体体现在以下几个方面：

（1）验证功能正确性：确认各可视化接口返回的数据与数据库实际数据一致，预测接口能够正确调用模型并返回合理的预测概率。

（2）评估模型性能：通过在独立测试集上计算AUC、精确率、召回率等指标，客观评估模型的预测能力，确保模型在未见数据上具有良好的泛化性能。

（3）验证接口稳定性：对各API接口进行正常请求和异常请求的测试，确认接口在边界条件下的处理逻辑正确，不会因非法参数导致系统崩溃。

（4）评估用户体验：对前端页面的展示效果、交互响应速度和浏览器兼容性进行测试，确保用户能够流畅地使用系统的各项功能。

### 5.2 测试目标

本次系统测试的目标主要包括以下几项：一是确保所有可视化图表能够正确渲染并展示真实数据；二是验证预测模块在平销期和促销期两个场景下的预测结果均符合预期精度要求（AUC≥0.78）；三是确认系统在正常并发访问条件下接口响应时间满足要求；四是验证系统在Chrome、Edge、Firefox等主流浏览器上的兼容性。
